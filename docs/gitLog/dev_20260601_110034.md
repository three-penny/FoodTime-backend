## 变更详情
1. 新增 `app/utils/email_utils.py`，包含：
   - `send_verification_code_email()`: 生成6位随机数字验证码，通过 SMTP 发送到用户邮箱，以线程安全的内存 dict 存储（含过期清理机制）
   - `verify_code()`: 校验验证码是否正确且未过期，校验后自动删除
2. `auth_service.py` 移除 `DEFAULT_VERIFICATION_CODE = '000000'`，注册时改为调用 `verify_code()`，发送改为调用 `send_verification_code_email()`
3. `config.py` 的 `Config` 基类中新增 MAIL_SERVER、MAIL_PORT、MAIL_USE_SSL、MAIL_USE_TLS、MAIL_USERNAME、MAIL_PASSWORD、MAIL_DEFAULT_SENDER、VERIFICATION_CODE_EXPIRE_SECONDS 共8项配置
4. `.env-template` 列出所有需填写的环境变量（含邮箱 SMTP），供开发者复制为 `.env` 后填写

## 影响范围
- 影响模块：用户认证（注册与验证码发送）
- 影响接口：POST /api/v1/auth/send-code、POST /api/v1/auth/register
- 向前兼容：否，不再接受 000000 作为验证码，必须通过邮箱接收真实验证码
- 启动前提：需在 `.env` 中正确配置 SMTP 信息

## 合并冲突处理指引

- 若 `config.py` 发生冲突，保留 MAIL_* 和 VERIFICATION_CODE_* 配置项
